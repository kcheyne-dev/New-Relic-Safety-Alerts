Just a preview of how the identity chip could badge each access level. Real role assignment will come from your auth provider (Okta). Admin = green (full system trust), CMT Member = blue (incident command), Office Manager = yellow (local awareness), Employee = grey (read-only / self-status).

Behavior summary

  • Identity chip — passive, not clickable. Sourced from auth (Okta) when wired up. Today: hardcoded for the prototype.
  • Open Incidents — clickable → opens Incidents panel filtered to Open. Red when ≥1 incident has Extreme severity, orange when there's any open, muted when zero.
  • Highest Active — clickable → zooms map to that alert + opens Alert Details. Color tracks severity. Title truncates first on narrow screens.
  • Need Help — clickable → jumps to incident with most Help responses, opens Responses tab filtered to Help.
  • Sources — clickable → opens the existing Freshness modal. Shows X/Y. Green dot when all healthy, yellow when any stale, red when any source errored.
  • Last updated — passive. Shows relative time of last data sync. Pulsing dot indicates recent save (persistence layer).
  • Critical state wash — when any Extreme alert is active, the strip gets a subtle red gradient from the left edge. Designed for peripheral attention without dominating the screen.
